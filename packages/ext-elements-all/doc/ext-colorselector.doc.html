alphaDecimalFormat
The format used by {@link Ext.util.Format#method!number Ext.util.Format#number} to format the alpha channel's value.

cancelButtonText
Text value for "Cancel" button; these are relied upon by the {@link Ext.ux.colorpick.Button} and the {@link Ext.ux.colorpick.Field}.

color
This config property is used internally by the UI to maintain the full color. Changes to this config are automatically reflected in `{@link #cfg!value value}` and vise-versa. Setting `{@link #cfg!value value}` can, however, cause the alpha to be dropped if the new value does not contain an alpha component.

fieldPad
padding between the sliders and HEX/R/G/B fields.

fieldWidth
Width of the text fields on the container (excluding HEX); since the width of the slider containers is the same as the text field under it (it's the same vbox column), changing this value will also affect the spacing between the sliders.

format
The color format to for the `{@link #cfg!value value}` config. The `{@link #cfg!value value}` can be set using any supported format or named color, but the stored value will always be in this format. Supported formats are: - hex6 - For example "ffaa00" (Note: does not preserve transparency). - hex8 - For eaxmple "ffaa00ff" - the last 2 digits represent transparency - #hex6 - For example "#ffaa00" (same as "hex6" but with a leading "#"). - #hex8 - For example "#ffaa00ff" (same as "hex8" but with a leading "#"). - HEX6 - Same as "hex6" but upper case. - HEX8 - Same as "hex8" but upper case. - #HEX6 - Same as "#hex6" but upper case. - #HEX8 - Same as "#hex8" but upper case.

okButtonText
Text value for "Ok" button; these are relied upon by the {@link Ext.ux.colorpick.Button} and the {@link Ext.ux.colorpick.Field}.

showOkCancelButtons
Whether Ok and Cancel buttons (in upper right, below the selected color preview) should be shown; these are relied upon by the {@link Ext.ux.colorpick.Button} and the {@link Ext.ux.colorpick.Field}.

showPreviousColor
Whether "previous color" region (in upper right, below the selected color preview) should be shown; these are relied upon by the {@link Ext.ux.colorpick.Button} and the {@link Ext.ux.colorpick.Field}.

value
The initial color to highlight; see {@link #cfg!format #format} for supported formats.
