compact
`true` to display this view in compact mode, typically used for smaller form factors. Setting to `true` applies any configured {@link #cfg!compactOptions #cfg-compactOptions}. <br>The compact config is applied to all configured {@link #cfg!views #cfg-views}.

compactOptions
A series of config options for this class to set when this class is in {@link #cfg!compact #cfg-compact} mode. <br>The compactOptions config is applied to all configured {@link #cfg!views #cfg-views}.

defaultView
The key of the item from {@link #cfg!views #views} to use as the default.

layout
Configuration for this Container's layout. Example: @example Ext.create({ xtype: 'container', layout: { type: 'hbox', align: 'middle' }, items: [{ xtype: 'panel', flex: 1, bodyStyle: { background: "#000", color:"#fff" } }, { xtype: 'panel', flex: 2, bodyStyle: { background: "#f00", color:"#fff" } }] });

store
A {@link Ext.calendar.store.Calendars} instance or configuration <br>The store config is applied to all configured {@link #cfg!views #cfg-views}.

timezoneOffset
The timezone offset to display this calendar in. The value should be specified in the same way as the native Date offset. That is, the number of minutes between UTC and local time. For example the offset for UTC+10 would be -600 (10 hours * 60 minutes ahead). Defaults to the current browser offset. <br>The timezoneOffset config is applied to all configured {@link #cfg!views #cfg-views}.

value
The value for the current view. value: new Date('10-02-2016') // to set the date to Oct 2nd 2016 <br>The value config is applied to all configured {@link #cfg!views #cfg-views}.

views
The calendar views to have available, each item in this configuration (labelled by a key) is to contain the configuration for the view, a class that extends {@link Ext.calendar.panel.Base}. Example with a day and week view: views: { day: { xtype: 'calendar-day' }, week: { xtype: 'calendar-week' } } The "day" and "week" keys would be the eligible values for the {@link #cfg!defaultView #cfg-defaultView} and the param string to pass to {@link #method!setView #method-setView}.
